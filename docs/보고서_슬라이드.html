1. 소개 (Introduction)

배경 및 문제 제기

  • 의약품에는 함께 복용하면 위험한 병용금기 조합이 존재하며, 국내에서도 식약처·HIRA가 다수의 금기 조합(약 1,628쌍)을 관리합니다.
  • 금기 확인의 선행 과제는 사진 속 개별 약품을 정확히 식별하는 것입니다. 그러나 한 장에 약품 4개가 다양한 각도로 찍혀 있어, 그대로 다루면 객체 검출(object detection) 문제가 되어 수업 범위를 벗어납니다.

가설 및 목표

  • 가설 · 다중 약품 사진을 bbox로 크롭해 단일 약품 이미지로 만들면 CNN 분류 문제로 환원할 수 있고, 분류 결과를 성분명으로 변환해 금기 DB와 대조하면 자동 검출이 가능하다.
  • 목표 · 약품 100종을 분류하는 CNN을 구현하고(128×128 입력), 분류 결과를 병용금기 조회 파이프라인과 연결해 위험한 조합을 자동으로 경고하는 end-to-end 시스템을 완성한다.

2. 방법 (Methods) — 데이터 및 전처리

데이터 설명

  • 원천 · 약품 사진 약 30GB(원본 976×1280), 사진당 약품 4종 + 다중 각도
  • 주석 · COCO 포맷 JSON 라벨 46,117개, 약품별 bounding box
  • 고유 약품코드 · 118종 → 100종 학습 클래스로 선정
  • 전처리 결과 · bbox 크롭으로 단일 약품 43,350장

전처리 로직

  • 직사각형 bbox 크롭 · 정밀 외곽선(누끼)은 분류 기여가 작아 생략
  • 리사이즈·정규화 · 128×128 이중선형 보간 후 ImageNet 통계로 픽셀값 정규화. ImageNet 통계 = 대형 이미지셋의 RGB 채널별 평균·표준편차(μ=0.485/0.456/0.406)로, (x−μ)/σ 변환 시 분포가 평균 0·표준편차 1에 맞춰져 수렴이 빨라짐. 입력을 일정한 크기·분포로 맞추는 학습 전 표준 전처리이며, 모델 내부의 BatchNorm과는 별개의 단계.
  • 사진(combo) 단위 분할 · 같은 사진의 각도 변형이 train·검증에 섞이는 누수를 막기 위해, 한 사진의 모든 크롭을 train/val/test 한 곳에만 배정 → 70/15/15, 분할 간 공유 사진 0건.
100종 모두 학습에 사용 · 클래스별 159~1,098장으로 불균형 존재

2-3. 방법 (Methods) — 아키텍처 설계 근거

각 구조 결정은 '식별 정확도 ↔ 학습 비용'의 균형과, 파라미터가 FC1에 집중되는 특성을 고려해 정했습니다.

설계 결정선택근거
입력 해상도128×128약품 각인·글자를 식별하기에 충분하면서 ImageNet급(224)보다 작아 1~2시간 학습 예산에 맞춘 해상도.
네트워크 깊이Conv 블록 4개MaxPool이 절반씩 줄여 128→64→32→16→8. 5블록(4×4)은 정보 과소실, 3블록(16×16)은 FC 입력 폭증 → 4블록이 균형점.
채널 수32→64→128→256해상도가 줄어드는 만큼 채널을 2배씩 늘려 표현력(정보량)을 유지하는 피라미드 구조.
커널·패딩3×3, pad=1, stride=1작은 커널을 쌓아 적은 파라미터로 넓은 수용영역 확보. 합성곱은 크기를 보존하고 축소는 Pool이 전담.
Dropout 위치FC1 뒤 · p=0.5전체 8.83M 중 FC1이 약 8.39M(95%) — 과적합 위험이 가장 큰 층을 정조준해 정규화.
정규화BatchNorm (각 conv 뒤)층별 입력 분포를 안정화해 수렴을 가속하고 그래디언트를 안정시킴.

→ 핵심은 파라미터의 약 95%가 FC1에 몰린다는 점입니다. 그래서 Dropout을 정확히 그 지점에 두고, 합성곱부는 작은 커널·풀링으로 가볍게 유지했습니다.

2-3-1. 방법 (Methods) — 설계 근거 심화

앞 장의 구조 결정에 더해, 메모리·해석 관점의 설계 근거를 '입증된 값'과 '관례에서 따라온 값'으로 구분해 정리했습니다.

쟁점핵심 근거
채널을 2의 거듭제곱으로
(32·64·128·256)		 GPU·CPU의 메모리 접근과 벡터 연산은 2의 거듭제곱 크기에서 정렬(alignment)·타일링이 맞아, 같은 파라미터 수라도 텐서 연산 처리량과 캐시 효율이 높습니다. 임의 값(50·100)보다 최적화된 커널 경로를 타기 쉽습니다.
8×8의 성격
(증명된 최적 아님)		 최종 8×8은 실험으로 최적을 입증한 값이 아니라 "입력 128 + 블록 4개"의 산술적 귀결(128→64→32→16→8)입니다. 6×6·16×16과의 비교는 수행하지 않았으며 향후 과제로 둡니다.
각 채널이 학습하는 특징
(해석상 추정)		 얕은 층(32·64)은 에지·색 대비·질감 등 저수준 보편 특징, 깊은 층(128·256)은 이를 조합한 윤곽·각인·색+형태 등 고수준 특징에 반응하는 경향. 단, 채널 1개가 개념 1개에 일대일 대응하지는 않습니다.
