1. 소개

◆ 배경

의약품은 함께 복용하면 안 되는 병용금기(竝用禁忌) 조합이 존재하며, 국내에서도 식약처·HIRA가 다수의 금기 조합을 관리한다. 본 프로젝트는 이러한 금기 조합 1,628쌍(엑셀 DB)을 활용하되, 사용자가 약품의 코드명을 일일이 입력하지 않고 사진만으로 약품을 식별한 뒤 자동으로 금기 여부를 확인할 수 있게 하는 것을 출발점으로 삼았다.

◆ 문제 제기

병용금기 확인의 선행 과제는 사진 속 개별 약품을 정확히 식별하는 것이다. 한 장의 사진에 여러 약품이 함께 찍히고 촬영 각도도 제각각이므로, 이를 그대로 다루면 객체 검출(object detection) 문제가 되어 수업 범위를 벗어난다. 따라서 문제를 개별 약품 이미지 분류(classification) 로 재정의할 필요가 있었다.

◆ 가설

  1. 다중 약품 사진을 bbox 좌표로 크롭하여 단일 약품 이미지로 전처리하면, CNN 분류 문제로 환원할 수 있다.
  2. 수업에서 학습한 표준 CNN 구조(Conv–BN–ReLU–Pool 반복 + FC)만으로도 약품 100종을 실용적 수준으로 분류할 수 있다.
  3. 분류 결과를 성분명으로 변환해 병용금기 DB와 대조하면, 금기 조합 자동 검출 파이프라인을 구성할 수 있다.

◆ 목표

  • 약품 100개 클래스를 대상으로 한 CNN 이미지 분류기 구현 및 학습
  • 학습된 분류기를 병용금기 DB 조회 파이프라인과 연결해 end-to-end 흐름 설계
  • 정확도 극대화보다 수업에서 배운 CNN 원리(전방향 전파·역전파·정규화·조기 종료 등)의 적용을 입증

2. 방법

◆ 데이터 설명

정규화 ImageNet 통계 (μ=0.485/0.456/0.406, σ=0.229/0.224/0.225)
학습/검증/테스트 분할 사진(combo) 단위 그룹 분할 70/15/15, seed 42 (0_split.py)

전처리에서 직사각형 bbox 크롭을 채택했다. 약품의 정밀 외곽선 추출(누끼)은 분류 정확도 향상에 기여가 작은 반면 비용이 커, CNN 분류에는 직사각형 크롭으로 충분하다고 판단했다. 정규화에 ImageNet 통계를 사용한 이유는, 약품도 실세계 색상을 가지므로 사전학습 통계가 수렴에 유리하기 때문이다.

ImageNet 통계란. ImageNet(약 120만 장 규모의 대형 이미지 데이터셋) 전체 픽셀값을 RGB 채널별로 미리 구해 둔 평균(μ)과 표준편차(σ)로, 사실상 "자연 이미지 픽셀의 대표 분포"를 요약한 값이다. 입력 이미지를 채널별로 x_norm = (x − μ) / σ 로 변환하면 각 채널 분포가 평균 0·표준편차 1 근처로 맞춰져, 학습 초기에 값의 크기가 들쭉날쭉하지 않아 경사하강이 안정적이고 수렴이 빨라진다. 약품도 실세계 색을 가진 자연 이미지이므로 별도 통계를 새로 구하지 않고 검증된 ImageNet 통계(μ=0.485/0.456/0.406, σ=0.229/0.224/0.225)를 그대로 적용했다. 이는 입력을 일정한 분포로 맞추는 학습 전 표준 전처리이며, 모델 내부에서 미니배치마다 분포를 정규화하는 BatchNorm과는 별개의 단계다.

COCO 형식이란. 라벨 JSON은 객체 검출·분할용 표준 주석 형식인 COCO(Common Objects in Context) 포맷을 따른다. 핵심은 각 약품의 위치를 "bbox": [x_min, y_min, width, height] 네 값으로 표기하는 방식으로, 각각 박스 좌상단의 x·y 픽셀 좌표와 너비·높이를 뜻한다. 본 프로젝트는 이 좌표로 원본 사진에서 I[y_min:y_min+h, x_min:x_min+w] 영역을 잘라 개별 약품 이미지를 생성했다. 즉 COCO bbox가 "검출 문제 → 분류 문제" 전환의 출발점이다(정밀 외곽선을 담는 segmentation 필드는 사용하지 않았다).

분할 방식(중요). 한 장의 원본 사진(combo)에는 약 3~4종이 함께 찍혀 있고, 같은 사진을 각도·조명만 바꿔 촬영한 변형이 사진 하나당 평균 3장 존재한다(전역 고유 사진 3,988장 → 크롭 43,350장). 따라서 이미지를 무작위로 나누면 거의 동일한 변형이 train과 검증에 동시에 들어가, 모델이 약이 아니라 사진의 배경·조명을 외우는 데이터 누수가 생긴다. 이를 막기 위해 파일명의 사진 식별자(combo)를 그룹 키로 삼아 한 사진의 모든 크롭을 train/val/test 중 한 곳에만 배정했다. 그 결과 train 30,444장 / val 6,479장 / test 6,427장으로 나뉘었고, 세 분할 사이에 공유되는 사진은 0건이며 100개 클래스가 세 분할에 모두 존재한다. test는 학습·검증에 일절 사용하지 않고 최종 평가에만 쓴다.

◆ 학습 클래스 선정 기준

고유 약품 코드 118종 중 100종을 다음 우선순위로 선정했다.

  1. 병용금기 쌍에 등장하는 약물 우선 포함 — 프로젝트 최종 목표가 금기 조합 검출이므로, 데이터셋 내 금기 약물(아질렉트정 등 3종 확정)을 반드시 학습 클래스에 포함.
  2. 나머지는 데이터 수량 기준 상위 클래스 선택 — 크롭 이미지 수가 많은 약물 순으로 채워 학습 안정성을 확보.

왜 100개로 나눴나(118종 중 18종 제외). 클래스 수를 100으로 고정한 것은 두 가지 때문이다. 첫째, 크롭 수가 극히 적은 하위 클래스를 포함하면 해당 클래스의 학습·검증 표본이 부족해 정확도 추정이 불안정해지고 불균형이 심화된다. 둘째, 100이라는 라운드 넘버는 클래스 불균형 비율·Macro F1 등 지표를 보고하고 모듈을 설계할 때 기준점으로 다루기 쉽다. 따라서 금기 약물을 먼저 확보한 뒤, 데이터가 충분한 상위 클래스로 100을 채우고 표본이 빈약한 18종은 제외했다. 이는 분류기와 최종 검출 목표를 일관되게 연결하는 선택이다.

◆ DL 모델 선정 이유

  • CNN 선정 근거: 입력이 영상 데이터이며, 합성곱은 지역적 패턴(글자·색·형태)을 가중치 공유로 효율적으로 학습하고 평행이동에 강건하다. 약품 식별은 표면의 각인·색·형태 같은 국소 특징에 의존하므로 CNN이 적합하다.
  • 구조 — DrugCNN: [Conv(3×3, pad=1) → BatchNorm → ReLU → MaxPool(2×2)] 블록 4개를 쌓아 점진적으로 추상적 특징을 추출하고, FC + Dropout으로 분류한다. 수업에서 다룬 표준 CNN 블록 구성(CONV→ReLU→POOL 반복)을 그대로 따랐다.
  • 채널·해상도 변화: (3,128,128) → (32,64,64) → (64,32,32) → (128,16,16) → (256,8,8) → Flatten(16,384) → FC1(512)+Dropout → FC2(100)
  • 규모: 약 8.83M 파라미터 (대부분이 FC1의 16,384×512에 집중, 최종 가중치 파일 33.7MB).

◆ 아키텍처·설정 설계 근거 (심화)

위 구조와 설정에서 자주 나오는 "왜 그렇게 정했나"를 정리한다. 일부는 실험으로 입증한 값이고, 일부는 설계 관례에서 따라 나온 값이라 그 성격을 구분해 기술한다.

① 블록을 4개로 정한 근거. 블록 수는 별도 비교 실험이 아니라 입력 해상도와 산술적으로 맞물려 결정됐다. 블록마다 MaxPool(2×2)이 가로·세로를 절반으로 줄이므로, 입력 128을 2로 4번 나누면 정확히 128 → 64 → 32 → 16 → 8이 된다. 4단계면 얕은 층의 저수준 특징(획·테두리·색)에서 깊은 층의 고수준 특징(약품 형태·각인 패턴)까지 약품 식별에 필요한 추상화 수준에 도달한다. 더 깊게 쌓으면 특징맵이 4×4·2×2로 과도하게 작아져 공간 정보가 뭉개지고, 덜 쌓으면 Flatten 차원이 커져 분류기 파라미터가 급증한다.

② 8×8의 성격(증명된 최적값이 아님). 최종 특징맵 8×8은 "가장 이상적임을 실험으로 증명한 값"이 아니라 "입력 128 + 블록 4개"를 정하면 자동으로 따라 나오는 값이다. 다른 크기(6×6, 16×16 등)와의 비교 실험은 수행하지 않았다. 다만 8×8은 공간 정보 보존과 Flatten 차원(256×8×8=16,384)의 크기 사이에서 균형이 무난한 지점이라는 정성적 판단은 성립한다. 향후 6×6·8×8·16×16을 비교하면 이 선택에 정량 근거를 추가할 수 있다.

③ 채널 수 규칙과 메모리 근거(32→64→128→256). 채널 수는 "해상도가 줄면 채널을 늘려 표현력을 보상한다"는 표준 설계 관례를 따랐다. MaxPool로 공간 원소가 1/4로 줄 때 채널을 2배로 늘리면 정보량 감소를 채널(특징 종류)의 다양성으로 메운다. 얕은 층은 저수준 특징이라 종류가 적어 32로 충분하고, 깊어질수록 특징 조합이 폭증해 더 많은 채널(256)이 필요하다. 또한 32·64·128·256처럼 2의 거듭제곱을 쓰는 이유는 메모리·연산에 유리하기 때문이다. GPU·CPU의 메모리 접근과 벡터 연산은 2의 거듭제곱 크기에서 정렬(alignment)·타일링이 잘 맞아, 같은 파라미터 수라도 텐서 연산 처리량과 캐시 효율이 좋다. 임의의 값(예: 50, 100)보다 커널 구현이 최적화된 경로를 타기 쉽다.

단계 해상도 채널 특징맵 원소 수
Block 1 64×64 32 131,072
Block 2 32×32 64 65,536
Block 3 16×16 128 32,768
Block 4 8×8 256 16,384

④ 채널 증가와 과적합의 관계, 그리고 통제 논리. 채널 증가는 곧 파라미터 증가이므로 원리상 과적합 위험을 키운다. 다만 본 모델 파라미터(8.83M)의 대부분은 Conv 채널이 아니라 FC1(16,384×512 ≈ 840만)에 집중돼 있어, 과적합의 주된 출처는 FC 층이다. 오히려 채널을 키워 8×8까지 압축한 덕분에 FC 입력 차원(16,384)이 그 선에서 억제된 면도 있다. 과적합은 다음 장치로 통제했다: FC1 뒤의 Dropout(0.5), 채널 분포를 정규화하는 BatchNorm, 회전·색 변화 데이터 증강, 그리고 조기 종료. 실제로 검증 정확도 99.18%에서 학습·검증 격차가 크게 벌어지지 않아, 이 설정에서 과적합은 통제된 것으로 본다(§3).

⑤ FC(완전연결층)의 역할. Conv가 이미지를 지역적으로 본다면, FC(Fully Connected)는 모든 특징을 종합해 최종 판단을 내리는 층이다. Conv 블록이 추출한 특징맵을 1차원으로 펼친 16,384개 값을 받아, FC1(16,384→512, +ReLU+Dropout)이 특징을 압축·종합하고 FC2(512→100)가 각 약품 클래스의 점수(로짓)를 출력한다. 모델 파라미터 대부분이 여기에 몰리는 것은 FC1의 16,384×512 연결 때문이다.

⑥ 각 채널이 학습하는 특징(해석상 추정). 본 프로젝트에서 특정 채널이 어떤 패턴에 반응하는지 feature map 시각화로 직접 확인하지는 않았으므로, 아래는 CNN 일반 원리에 근거한 추정이다. 얕은 층(Block 1~2, 32·64채널)의 채널은 대체로 방향성 에지·색 대비·단순 질감 같은 저수준 보편 특징에 반응하고, 깊은 층(Block 3~4, 128·256채널)의 채널은 이를 조합한 고수준·과제 특화 특징(알약의 둥근 윤곽, 캡슐 이음새, 표면 각인 패턴, 색+형태 조합)에 반응하는 경향이 있다. 다만 채널 하나가 사람이 이름 붙일 개념 하나에 일대일로 대응하지는 않으며, 여러 채널이 분산·중첩해 한 개념을 표현하는 경우가 많다. best_model.pth로 feature map을 시각화하면 이 추정을 관찰로 전환할 수 있다(향후 과제).

⑦ 조기 종료 시점(patience=8)의 근거. 조기 종료는 "검증 손실이 8에폭 연속 개선되지 않으면 중단"하는 규칙이다(개선 기준은 일관성을 위해 항상 일반 CE로 측정). patience=8 역시 격자 탐색으로 최적화한 값이 아니라, 두 위험 사이의 균형을 잡은 관례적 설정이다. 값이 너무 작으면 손실이 일시적으로 정체하는 평탄 구간에서 성급히 멈춰 과소적합의 위험이 있고, 너무 크면 검증 손실이 이미 악화된 뒤에도 학습을 끌어 과적합과 연산 낭비가 생긴다. 본 학습은 lr=3e-4에서 검증 정확도가 epoch 2에 이미 99%에 도달할 만큼 수렴이 빨라, 8에폭의 여유는 평탄 구간을 통과시키되 과적합 진입 전에 멈추기에 충분했다.

◆ 파라미터 결정 과정 (2단계 하이퍼파라미터 탐색)

스케일에 잘 전이되는 값(learning rate, batch size, dropout, augmentation) 과 전이되지 않는 값(epoch 수, 조기 종료 임계, 클래스 수 민감 설정) 을 구분하고, 다음 2단계로 진행했다.

  • 1단계 — 후보 선별: 20클래스 · 20에폭으로 빠르게 비교해 나쁜 후보 제거
  • lr=1e-3 → 최고 val_acc 99.21% (epoch 17)
  • lr=3e-4 → 최고 val_acc 99.37% (epoch 6·9), epoch 2에 이미 99% 도달 → 채택
  • lr=1e-4 → 20에폭 기준 과소수렴 우려로 제외
  • 2단계 — 본 학습: 확정한 lr=3e-4, batch=128, dropout=0.5로 100클래스 학습

학습 속도 또한 최적화했다. 초기 병목은 GPU가 아니라 이미지 파일 I/O였으며, num_workers=4 + persistent_workers=True로 데이터 로딩을 병렬화해 에폭당 약 13초로 단축했다.

3. 결과

◆ 사용한 평가지표와 선정 이유

최종 평가는 독립 test split 에서만 수행하며(5_evaluate.py), 정확도 한 가지에 의존하지 않도록 지표를 확장했다.

  • Top-1 / Top-5 Accuracy: Top-1은 최종 예측의 정확도, Top-5는 정답이 상위 5개 후보 안에 드는 비율이다. 약품처럼 외형이 비슷한 클래스가 많을 때 Top-5는 "거의 맞힌" 정도를 보여 준다.
  • Macro F1-score: 클래스별 F1을 단순 평균한 값. 클래스 불균형(7.0배) 환경에서 다수 클래스에 가려지는 소수 클래스 성능을 동등하게 반영하므로, 정확도보다 일반화 성능을 정직하게 보여 준다(Weighted F1도 함께 산출).
  • 혼동행렬(Confusion Matrix): 어떤 약품 쌍이 서로 헷갈리는지 100×100 행렬로 시각화해, 외형이 유사한 약품의 오분류 패턴을 진단한다.
  • 검증 손실(Validation Loss): 학습 중 best 모델 저장 및 조기 종료 기준으로 사용한다.

4. 결론

◆ 결과 요약

수업에서 학습한 표준 CNN(Conv–BN–ReLU–Pool 4블록 + FC + Dropout)으로 약품 100종 분류기를 구현했고, 2단계 하이퍼파라미터 탐색으로 lr=3e-4를 선정해 검증 정확도 99.18% 를 달성했다. 객체 검출 문제를 전처리 크롭을 통한 분류 문제로 재정의한 것이 프로젝트를 수업 범위 안에서 성립시킨 핵심 결정이었다.

◆ 인사이트

  • 문제 재정의의 힘: 모델을 키우기 전에 문제 자체를 분류로 환원한 것이 가장 큰 성능·효율 이득을 냈다.
  • 병목은 종종 연산이 아니라 I/O: GPU 활용도를 높인 것은 모델 변경이 아니라 데이터 로딩 병렬화였다.
  • 지표의 함정 이해: dropout으로 인한 train/val 지표 역전, val_loss 기준 best 모델 저장 등 수업 개념이 실제 로그 해석에 직접 적용됐다.

각 개선은 코드로 구현했고, 학습이 필요 없는 항목은 실제 수치로 검증했다.

◆ 5-1. 데이터 누수 차단 + 독립 테스트셋 (0_split.py)

문제의 핵심은 분할 단위였다. 같은 사진(combo)을 각도·조명만 바꿔 찍은 변형이 사진당 평균 3장이고, 한 사진의 크롭들은 서로 다른 클래스 폴더로 흩어진다(사진당 평균 3.64개 클래스). 무작위 분할은 이 변형들을 train·val에 흩어 배경·조명을 외우게 만든다.

해결은 파일명의 사진 식별자(combo)를 그룹 키로 삼아, 한 사진의 모든 크롭을 train/val/test 중 한 곳에만 전역 배정하는 것이다. 클래스가 세 분할에 모두 포함되도록 coverage를 검증한다.

test split은 학습·검증에서 완전히 격리되며, 최종 평가에만 사용한다.

◆ 5-3. 클래스 불균형 보정 (imbalance.py, 3_train.py)

클래스별 이미지 수는 156~1,086장으로 최다/최소 7.0배 차이가 난다. 세 가지 표준 기법을 구현하고 플래그로 전환한다.

  • 가중 CrossEntropyLoss(기본): 클래스 빈도의 역수를 가중치로 부여해 소수 클래스의 손실 기여를 키운다.
  • Focal Loss: (1−p_t)^γ 로 쉬운(다수) 샘플의 손실을 낮춰 어려운 소수 클래스에 집중한다(γ 기본 2.0).
  • Oversampling: WeightedRandomSampler로 소수 클래스 샘플을 더 자주 추출해 배치 균형을 맞춘다.

소수 클래스 성능은 5-2의 Macro F1·클래스별 F1로 확인한다.

◆ 5-4. 금기 검출 검증 강화 (validate_contraindication.py)

성분 매칭을 부분문자열에서 성분 단위 토큰 부분집합 매칭으로 교체하고, 직접 라벨링한 검증셋으로 정량 평가했다. 기존 방식은 ephedrine↔pseudoephedrine, omeprazole↔esomeprazole, ibuprofen↔dexibuprofen 같은 거짓 양성을 냈으나, 개선 방식은 검증셋 16케이스 전부를 맞혔다(Precision·Recall·Accuracy 모두 0.625→1.000). 상세 수치는 3절 표 참조.

◆ 5-5. 남은 한계와 향후 과제

  • 금기 조합의 희소성: 100개 학습 클래스 안의 실제 금기는 2쌍(약물 3종)뿐이다. 금기 약물을 더 포함하도록 데이터·클래스를 확장하면 검출의 실증 가치가 커진다.
  • 성분명 동의어: 토큰 매칭도 aspirin↔acetylsalicylic acid처럼 표기 자체가 다른 동의어는 잡지 못한다. 동의어 사전을 더하면 재현율이 더 오른다.
  • 재학습 기반 재측정: 5-2의 test 지표를 채워 누수 제거 전후의 정확도 변화를 정량 비교하는 것이 다음 단계다.

C. 파이프라인 파일 구성

  • main.py — 엔트리포인트 (preprocess / train / predict 서브커맨드)
  • 0_split.py — 사진(combo) 단위 그룹 분할로 data/splits.json 생성(누수 0건 검증)
  • 1_preprocess.py — bbox 크롭으로 단일 약품 이미지 생성 (클래스별 폴더 {idx:03d}_{code}/)
  • 2_model.py — DrugCNN 정의 (Conv–BN–ReLU–Pool ×4 + FC + Dropout)
  • imbalance.py — 클래스 불균형 대응(가중 손실·focal loss·oversampling)
  • 3_train.py — splits.json의 train/val 학습 + 불균형 보정 + 조기 종료 + 손실 곡선
  • 4_predict_and_check.py — 추론 → class_map.json 성분명 → 병용금기 DB 조회
  • 5_evaluate.py — test 전용 최종 평가(혼동행렬·Macro F1·Top-1/Top-5)
  • examination/find_contraindicated.py — 데이터셋 × 금기 DB 교차 검색
  • examination/validate_contraindication.py — 금기 매칭 정확도 검증 + 성분 단위 토큰 매칭
