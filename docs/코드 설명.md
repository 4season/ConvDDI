## 1. 데이터 파이프라인 및 전처리

### 📁 [0_split.py](file:///Users/junho/PythonProjects/deeplearning_class/team_project_06/drug_classification/src/0_split.py)
* **목적:** 데이터 누수(Data Leakage)를 원천 차단하기 위한 **'사진(combo) 단위' 그룹 train/val/test 분할** 스크립트입니다.
* **배경:**
  * 동일 약품 조합(combo)을 각도·조명만 바꾸어 촬영한 복제 이미지가 여러 장 존재합니다.
  * 하나의 원본 이미지 내에 여러 약품이 함께 찍혀 있어, 크롭된 이미지들이 각각 다른 클래스로 흩어집니다.
  * 이미지를 임의로 섞어 분할하면 동일 사진의 다른 크롭본이 train/val/test에 나뉘어 들어가 배경이나 촬영 구도를 암기하는 데이터 누수가 발생합니다.
* **핵심 기능:**
  * `combo_key(file_name)`: 파일명에서 촬영 각도 등 변형 접미사를 분리해 원본 사진 식별자(combo key)를 추출합니다.
  * `gather_images(class_dirs)`: 크롭 이미지들을 수집하며, 어떤 combo가 어떤 클래스들을 포함하는지 기록합니다.
  * `assign_combos(combo_classes, ratios, rng)`: combo 단위로 학습/검증/테스트 세트에 전역 배정합니다. 배정 후 모든 클래스가 모든 split에 골고루 포함되는지 검증(coverage check)하며, 누락 시 시드를 조율해 재시도합니다.
  * 배정 후 split 간에 겹치는 combo가 없는지 정밀 누수 검증을 수행하고 `data/splits.json`으로 내보냅니다.

### 📁 [1_preprocess.py](file:///Users/junho/PythonProjects/deeplearning_class/team_project_06/drug_classification/src/1_preprocess.py)
* **목적:** COCO 포맷 bounding box 좌표를 사용해 원본 복합 이미지에서 개별 약품을 크롭(Crop)하고, 이를 $128 \times 128$ 해상도로 리사이즈하여 학습용 디렉터리 구조로 분류·저장합니다.
* **핵심 기능:**
  * `load_class_map()`: `class_map.json`에서 성분 정보와 클래스 인덱스 매핑을 로드합니다.
  * `safe_crop(img, bbox)`: COCO 형식 `[x_min, y_min, width, height]`를 이미지 경계 밖으로 벗어나지 않게 보정하여 안정적으로 크롭을 수행합니다.
  * `process_single_json(...)`: 개별 JSON 파일로부터 이미지 파일 경로를 조합해 이미지를 읽어 크롭 및 리사이즈 후 `data/cropped/{class_idx}_{drug_code}/` 폴더에 PNG로 저장합니다.
  * `run_preprocess()`: 전체 라벨링 세트(`TL_1` ~ `TL_8`)에 대해 병렬적으로 전처리를 실행하고 성공/오류 통계를 출력합니다.

## 2. 딥러닝 모델 아키텍처

### 📁 [2_model.py](file:///Users/junho/PythonProjects/deeplearning_class/team_project_06/drug_classification/src/2_model.py)
* **목적:** 약품 이미지 분류를 위한 **DrugCNN 모델 아키텍처**를 정의합니다.
* **구조:**
  * **입력:** $(Batch, 3, 128, 128)$ 크기의 3채널 컬러 이미지
  * **Feature Extractor (합성곱 층):** 4개의 `ConvBlock`으로 구성
    * `ConvBlock`: `Conv2d(kernel=3, padding=1, bias=False)` ➔ `BatchNorm2d` ➔ `ReLU` ➔ `MaxPool2d(2x2)`
    * 채널 변화: $3 \rightarrow 32 \rightarrow 64 \rightarrow 128 \rightarrow 256$
    * 해상도 변화: $128 \rightarrow 64 \rightarrow 32 \rightarrow 16 \rightarrow 8$
  * **Classifier (완전연결 층):** 합성곱 층의 출력을 펼쳐(Flatten) 클래스 분류를 수행
    * Flatten ➔ $256 \times 8 \times 8 = 16,384$ 차원 벡터 생성
    * `FC1(16384 -> 512)` ➔ `ReLU` ➔ `Dropout(p=0.5)` ➔ `FC2(512 -> 100)`
* **핵심 메소드:**
  * `forward(x)`: 네트워크 순전파를 수행해 최종 로짓(logits)을 반환합니다.
  * `predict_proba(x)`: 최종 출력에 Softmax를 적용하여 클래스별 예측 확률을 계산합니다.
  * `predict(x)`: 가장 높은 확률을 갖는 클래스 인덱스를 반환합니다.
  * `print_model_summary(...)`: PyTorch 네트워크 구조를 순회하며 각 레이어의 출력 텐서 셰이프와 파라미터 개수를 정밀 진단합니다. (총 파라미터 수: 약 8.83M)

## 3. 학습 및 불균형 제어

### 📁 [3_train.py](file:///Users/junho/PythonProjects/deeplearning_class/team_project_06/drug_classification/src/3_train.py)
* **목적:** 데이터 누수가 방지된 split 정보와 클래스 불균형 제어 옵션을 기반으로 DrugCNN을 학습시킵니다.
* **핵심 기능:**
  * `SplitDataset`: `splits.json`에서 파싱된 경로를 바탕으로 데이터를 로드합니다. GPU 환경의 경우 I/O 대기시간을 줄이기 위해 RAM에 이미지를 사전 캐싱하는 기능을 지원합니다.
  * `get_transforms()`: 훈련 데이터에 대해 가중 증강(RandomHorizontalFlip, RandomRotation, ColorJitter)을 적용하고 검증용에는 리사이즈 및 정규화만 가합니다.
  * `train_one_epoch(...)` & `evaluate(...)`: 에포크별 학습 및 검증 동작을 정의합니다. 과적합 방지를 위해 일반 CrossEntropyLoss 검증 손실 기준의 Early Stopping을 수행합니다.
  * `save_loss_curve(...)`: 학습 종료 후 에포크별 손실 및 정확도 곡선 차트(`loss_curve.png`)를 그립니다.

### 📁 [imbalance.py](file:///Users/junho/PythonProjects/deeplearning_class/team_project_06/drug_classification/src/imbalance.py)
* **목적:** 데이터셋 내의 최대 7배에 달하는 클래스 수량 불균형 문제를 해소하기 위한 전략들을 제공합니다.
* **핵심 기법:**
  1. **가중 손실함수 (Weighted Loss):**
     * `compute_class_weights(...)`: 클래스 빈도의 역수를 계산하여 가중치 벡터 $w_c = \frac{N}{C \times n_c}$를 생성합니다. 소수 클래스 이미지 분류 실패 시 손실에 더 무거운 벌점을 가합니다.
  2. **샘플링 가중치 (Oversampling):**
     * `make_sample_weights(...)`: 샘플별로 해당 클래스 빈도의 역수 확률로 가중치를 할당하여 PyTorch의 `WeightedRandomSampler`에서 소수 클래스가 배치에 균일하게 잡히도록 유도합니다.
  3. **Focal Loss:**
     * `FocalLoss(nn.Module)`: 이미 높은 신뢰도로 잘 분류해내고 있는 다수 클래스(쉬운 샘플)의 손실 기여도를 낮추고, 틀리기 쉬운 소수 클래스에 역량을 집중하는 손실 함수를 구현합니다.

## 4. 최종 평가 및 시각화

### 📁 [5_evaluate.py](file:///Users/junho/PythonProjects/deeplearning_class/team_project_06/drug_classification/src/5_evaluate.py)
* **목적:** 훈련 중 보지 못한 독립 `test split`만을 활용하여 모델의 최종 일반화 성능을 투명하게 평가합니다.
* **핵심 기능:**
  * **평가 지표:** Top-1 / Top-5 Accuracy 및 소수 클래스 탐지 성능 반영을 위한 Macro F1-score와 Weighted F1-score를 독립 계산합니다.
  * `confusion_matrix_np(...)`: 100개 클래스 간의 혼동행렬(Confusion Matrix)을 연산합니다. 외부 라이브러리(scikit-learn) 미설치 환경에서도 Numpy로 동등하게 처리되도록 안전장치를 구축했습니다.
  * `per_class_prf(...)`: 혼동행렬로부터 클래스별 정밀도(Precision), 재현율(Recall), F1 점수를 연산합니다. 성능 최저 5개 클래스를 정렬 출력하여 모델 취약점을 진단합니다.
  * `save_confusion_png(...)`: 행 기준 정규화(Row-normalized)를 적용한 오차 행렬 히트맵을 생성하여 오분류 경향성을 시각화합니다.

### 📁 [6_feature_maps.py](file:///Users/junho/PythonProjects/deeplearning_class/team_project_06/drug_classification/src/6_feature_maps.py)
* **목적:** 임의의 약품 이미지를 모델에 통과시킬 때 각 Conv 블록 내부 채널들의 Feature Map(특징 활성화 지도)을 저장합니다.
* **핵심 기능:**
  * `collect_feature_maps(model, x)`: PyTorch의 `register_forward_hook`을 활용해 중간 Conv 층 출력을 가로챕니다.
  * `plot_block(...)`: 채널별 최소-최대 정규화(Min-Max Normalization)를 통해 명암 대비를 강화한 뒤 각 Conv 블록의 채널 16개를 바둑판 형태로 배열하여 이미지로 저장합니다.
  * 이를 통해 얕은 층은 경계선이나 원형 윤곽, 색상 등에 강하게 활성화되고, 깊은 층은 약품 고유의 각인 문자나 구체적인 알약 형태 등 추상적 정보에만 활성화되는 전형적인 CNN의 시각적 판단 메커니즘을 규명합니다.
